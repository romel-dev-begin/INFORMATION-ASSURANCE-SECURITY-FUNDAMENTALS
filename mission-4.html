TOPIC-04 SECURE

Security Policies, Standards, and Procedures

01 Policy
A security policy is a formal rule that tells users what is allowed and not allowed when using information systems. It gives direction on how information and technology should be protected.

Examples of Security Policies:
• Password policy
• Acceptable use policy
• Data privacy policy
▸ POLICY EXAMPLE
"Users must create strong passwords and change them every 90 days."
02 Standard
A standard provides specific requirements that support a policy. While a policy gives the general rule, the standard gives the exact requirement.

A standard tells people the required level or format to follow.
▸ STANDARD EXAMPLE
Policy: Users must create strong passwords.
Standard: Passwords must have at least 8 characters, uppercase and lowercase letters, numbers, and special characters.
03 Procedure
A procedure is a step-by-step instruction on how to perform a security task. It explains how a policy or standard should be applied in real situations.
▸ PROCEDURE EXAMPLE
Password Reset Procedure:
1. User requests password reset.
2. System verifies user identity.
3. Temporary password or reset link is provided.
4. User creates a new password.
5. System records the activity in the log.
04 Policy vs Standard vs Procedure
Policy: The rule or instruction that must be followed.

Standard: The specific requirement that supports the rule.

Procedure: The step-by-step process for applying the rule.
📋
POLICY
The rule that defines what must be done and why. Sets high-level direction and intent.
WHY
📏
STANDARD
The specific requirement that makes the rule measurable and enforceable.
WHAT
📝
PROCEDURE
The step-by-step instructions that tell you exactly how to comply.
